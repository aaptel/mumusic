faq

What is the purpose of this site?

I find the music made by the people of /mu/ really interesting. It's usually shared in bandcamp or soundcloud threads where people post and comment music. Once the thread dies, everything is lost. The point of this website is to keep a trace of everything posted on those threads: the music and the comments.

Right now, only bandcamp threads are handled, but it might change.

When is the site updated?

The archive is updated every 30 minutes.

How does the thread detection work?

I've tweaked it for a long time to see what works. Right now to be archived a thread needs:

  • at least 3 posts containing bandcamp links.
  • a subject which doesn't contain sharethread.
  • a post or subject containing bandcamp followed by any of the following words: thread, topic, general, station. It doesn't have to be exactly after it, just in the rest of the subject/post.

What did you use to make this?

The software running the site is free (as in freedom). You can find the source code on github. Everyone is welcomed to improve it.

It's written in Python and runs on a free instance of Google App Engine. It's my first project using it so the code may be crappy/not scalable. I'm trying to improve it as I go.

I have a great idea! Why don't you do X?

You can make a feature request on github, where it can be discussed.
